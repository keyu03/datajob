// FEATURES

為資料工程師
量身設計的每一個細節

從 SQL 撰寫到跨庫同步，從單次執行到週期排程，DataJob 讓所有事情都在瀏覽器裡完成。

🔄
多來源連線同步
單一 Job 可選取多個來源連線，全部執行相同的 Source SQL，結果匯入同一個目標資料庫，自動記錄每個來源的執行明細。
multi-source per-source log
🗂️
Pipeline 流水線
將多個 Job 串成 Pipeline，依序執行。支援 on_failure 策略（stop / continue / skip），自動追蹤每步驟的狀態與耗時。
pipeline step log
⏰
靈活排程管理
支援 daily、weekly、monthly、cron 表達式、每 N 分鐘等多種排程類型，互動式設定介面，不需記憶 cron 語法。
APScheduler cron interval
🔁
雙層迴圈執行
支援查詢迴圈與日期迴圈獨立或巢狀組合，自動依公式生成日期序列。每次迭代可執行前置 SQL，適合分批同步大量歷史資料。
date loop query loop
📊
完整執行紀錄
每次執行自動記錄狀態、耗時、擷取／寫入列數、渲染後 SQL。多來源執行時分別記錄每個來源連線的詳細數據，支援篩選查詢。
execution log source detail
🛠️
視覺化 SQL 編輯器
內建 SQL 編輯器支援行號顯示、一鍵格式化、Jinja2 參數提示、來源資料即時預覽，Target DB 表格拖拉自動產生 INSERT 語法。
CodeMirror Jinja2 preview
// HOW IT WORKS

三步驟，資料自動流動

01
設定連線與 Job
在連線管理頁面建立來源與目標資料庫連線（支援密碼加密儲存）。新增 Transfer Job，選取一或多個來源連線，撰寫 Source SQL 與 Target SQL。
02
設定排程或 Pipeline
將 Job 加入 Pipeline 組成流水線，或直接設定排程（cron / daily / interval）。所有設定均在瀏覽器完成，無需接觸伺服器設定檔。
03
自動執行 · 追蹤結果
排程時間到來，系統自動執行並記錄完整執行明細。Dashboard 即時顯示最近執行狀況；任何失敗均有詳細錯誤訊息可追查。

寫 SQL，
其他交給我們

內建 Jinja2 模板引擎，{{ param }} 語法動態帶入參數。來源資料即時預覽，Target DB 表格拖拉自動生成帶 Named Bind 的 INSERT SQL。

  • 行號 gutter + 一鍵格式化（sqlparse）
  • Jinja2 未定義變數即時提示
  • Source SQL 資料預覽（前 N 筆）
  • Target DB 表格拖拉產生 INSERT 語法
  • Dry Run 語法驗證（支援 Oracle Named Bind）

每一次執行
都有跡可循

  • 主列顯示合計 + 子列顯示各來源明細
  • 渲染後的完整 SQL 供事後稽核
  • 連線 host / 資料庫 / 耗時一目瞭然
  • Pipeline 逐步驟展開查看
  • 支援依狀態與觸發方式篩選
